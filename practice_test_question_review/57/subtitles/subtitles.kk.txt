Қайырлы күн, құрметті студенттер! Біз Practice Test сұрағын талдауды бастаймыз. Сұрақтың шарты бойынша біз онлайн оқыту компаниясымыз. Біздің қолданба жергілікті дата-центрде Windows Server машиналарында орналастырылған, Microsoft.NET жүйесінде жазылған. Біз Oracle дерекқорының стандартты нұсқасын қолданамыз. Басшылық жоғары қолжетімді болу үшін IT инфрақұрылымын AWS бұлтына көшіруді шешті. Тасымалдау процесі кезінде код өзгерістерін азайтып, бұлттағы инфрақұрылымдық ортаңызды оңай басқара алуыңыз керек. Ұсынылған нұсқалардан осының барлығын жүзеге асыруға мүмкіндік беретін екі нұсқаны таңдау қажет. Ұсынылған бес жауап нұсқасын көресіз. Екінші және үшіншісі - дұрыс жауап нұсқалары. Жауап нұсқалары бір-бірінен өте ерекшеленетінін ескеріңіз, сондықтан бұл сұрақты орташадан жоғары қиындық деңгейі ретінде жіктеуге болады. Олай болса, әр жауап нұсқасын қарастырып, оның неліктен дұрыс немесе бұрыс екенін көрейік. Біз екі сервисті еске түсіруіміз керек. Біріншісі - AWS Database Migration Service, қысқаша AWS DMS. Бұл - жергілікті дерекқорды AWS бұлтына көшіруге мүмкіндік беретін сервис. Оның көптеген пайдалы функциялары бар, олардың бірі AWS Schema Conversion Tool немесе AWS SCT. Бұл функцияның арқасында дерекқордың қозғалтқышын өзгертуге болады. Мысалы, егер сізде жергілікті Oracle дерекқоры болса, онда сіз бұл құралды басқа қозғалтқыш үшін, мысалы, MySQL үшін ұқсас схеманы жасау үшін пайдалана аласыз және AWS бұлтындағы MySQL дерекқорымен жұмысты жалғастыра аласыз. Тағы бір сервис - AWS Elastic Beanstalk. Бұл – бұлтта инфрақұрылымды басқаруды жеңілдетуге мүмкіндік беретін сервис. Сізге тек бағдарламаңыздың, қолданбаңыздың кодын жүктеп салу керек, сонымен қатар кіріс ретінде бұлттағы болашақ инфрақұрылым үшін конфигурация файлын жіберу керек. Енді дұрыс емес жауап нұсқаларын қарастырайық. Контейнерлерді қамтитын екі жауап нұсқасы, Amazon EKS және Amazon ECS – дұрыс емес. Сұрақ шарты бойынша, әзірлеушілер тобының қатысуы аз болатын опцияны таңдау керек. Мұнда контейнерлер туралы айтатын болсақ, контейнерлерге ауысу қолданбаны айтарлықтай өзгертуді талап етеді, сондықтан осы екі нұсқа да дұрыс емес. AWS Application Migration Service сервисіне қатысты үшінші нұсқа да сәйкес келмейді. Бізде бөлек тасымалдануы қажет Oracle дерекқоры және бұлтқа бөлек тасымалдануы қажет қолданба бар. Қолданба жағдайында Elastic Beanstalk қолайлы, ал Oracle жағдайында дерекқорды тасымалдауға маманданған қолданба AWS DMS жақсырақ. Сондықтан AWS Application Migration Service нұсқасы дұрыс емес. Осымен біз Practice Test сұрағын қарастырдық. Назарларыңызға рахмет.
