Қайырлы күн, құрметті студенттер! Біз Practice Test сұрағын талдауды бастаймыз. Сұрақтың шарты бойынша, бізде Elastic Container Service сервисін пайдаланатын жоғары жүктеме жүйесі бар. Әрі қарай, ECS EC2 инстанстарына негізделген. Жүйенің өзі жоғары жүктелген, оқу және жазу операциялары көп. Пайдаланушылар бүкіл әлем бойынша таратылады және бір уақытта жүздеген ECS тапсырмалары көп жағдайда өңделеді. ECS тапсырмаларының шығысына келетін болсақ, бұл шамамен 10 МБ деректер. Бұл деректердің барлығы мұрағатталатынын және біраз уақыттан кейін кейбір бөлігі жойылатынын ескере отырып, барлығы 10 ТБ-тан аспайтын деректерді сақтауымыз керек. Енді Solutions Architect ретінде нұсқалардың ішінен ең қолайлы шешімді ұсынуымыз керек. Бұл слайдта сіз жауап нұсқаларын көресіз, сонымен қатар төртінші жауап нұсқасы дұрыс екенін көресіз. Барлық жауап нұсқаларын сұрыптауға тырысайық. Назар аударсаңыз, екінші және төртінші жауап нұсқалары өте ұқсас, олар тек EFS түрінің өзгеруімен ғана ерекшеленеді. Бірінші жағдайда, бұл Bursting Throughput mode, Performance Mode - General Purpose. Екінші жағдайда, Provisioned Throughput mode және Performance mode ретінде Max I/O қосу ұсынылады. Яғни, бірінші жағдайда бұл жалпы EFS параметрлері, ал екінші жағдайда бұл үлкен жазу және оқу операцияларын қажет ететін EFS параметрлері. Екі нұсқаның бірі дұрыс екені анық, бірақ соған қарамастан бірінші және үшінші жауап нұсқаларын қарастырайық. Біріншісіне қарасаңыз, онда Amazon FSx File Gateway туралы айтылады. Бұл Windows машиналары үшін Shared File System орнатуға көмектесетін сервис. Сұрақта операциялық жүйе туралы ештеңе айтпайды, біз оның Linux немесе Windows екенін білмейміз, сондықтан әрі қарай оқимыз. Бірінші жауапта Storage Gateway туралы айтылады. Мұнда сіз осыған назар аударуыңыз керек. Storage Gateway — бұлт пен on-premise жергілікті желіні қосуға көмектесетін сервис. Біздің жағдайда, on-premise туралы әңгіме болған жоқ, сондықтан біз бұл нұсқаны алып тастай аламыз. Үшінші нұсқа туралы айтатын болсақ, ол DynamoDB туралы. Назар аударсаңыз, ең соңында DynamoDB кестесін контейнерге mount point ретінде қосу ұсынылады. Негізгі Linux курсынан mount point файлдық жүйені Linux құрылғысына қосқан кезді айтатынын білеміз. DynamoDB файлдық жүйе емес, кесте, NoSQL дерекқоры болғандықтан, біз мұны істей алмаймыз. Сипаттаманың өзі қисынсыз болып көрінеді, сондықтан біз үшінші нұсқаны алып тастай аламыз. Бізде екінші және төртінші нұсқалар қалды. Мұнда деректер үнемі және жиі сұралатынын ескеріңіз. Сондай-ақ, бір уақытта жүздеген ECS тапсырмаларының көп бөлігі орындалатынын есте сақтаңыз. Сондықтан бұл өте жоғары жүктелген жүйелер. Және EFS үлкен I/O операцияларына арналған болуы керек деп болжау керек. Осыған байланысты төртінші нұсқа дұрыс. Сондай-ақ келесі слайдтарда мәтінді талдауды көресіз. Видеоны тоқтатып, асықпай қарап шықсаңыз болады. Осымен Practice Test сұрағын талдауымыз аяқталды. Назарларыңызға рахмет.
