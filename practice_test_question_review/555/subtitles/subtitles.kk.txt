Қайырлы күн, құрметті студенттер! Біз Practice Test сұрағын талдауды бастаймыз. Сұрақтың шарты бойынша, компанияның әлемнің әртүрлі бөліктерінде әртүрлі департаменттер бар. Компания бүкіл инфрақұрылымды IТ бақылауына алуды қалайды, барлық шығындарды көргісі келеді. Әлемнің әртүрлі бөліктерінде орналасқан департаменттердің көзқарасы бойынша олар өз ресурстарына әкімшілік бақылауды сақтап қалғысы келеді, сонымен қатар олардың ресурстары басқа департаменттердің ресурстарымен қиылыспауын талап етеді. Департаменттердің автономиясын сақтап қалу, бірақ орталық IТ басқару мүмкіндігін және барлық шығындардың бір жерде көрінуін қамтамасыз ету үшін не істеу керек? Ұсынылған жауап нұсқаларының ішінен екі дұрыс жауапты таңдауымыз керек. Бұл слайдта сіз жауап нұсқаларын көресіз, сонымен қатар екінші және бесінші дұрыс екенін көресіз. Соған қарамастан, жауап нұсқаларының әрқайсысына тоқталып, оларды егжей-тегжейлі талдап көрейік. Бірінші жауап туралы айтатын болсақ, департаменттердің әрқайсысы үшін бөлек availability zone құру, сондай-ақ екі availability zone арасындағы байланысты жақсарту үшін AWS Global Accelerator пайдалану ұсынылады. Бұл жауап мүлдем қисынсыз. Неліктен? Себебі, біріншісі- availability zone құруды ұсынғандықтан, шын мәнінде, аймақтар мен аймақтардағы availability zone AWS арқылы жасалады, біз бұған әсер ете алмаймыз. Екінші мәселе, availability zone-ның саны шектеулі және сіздің департаменттеріңізді бір аймаққа шексіз қоса беруге болмайды. Тағы бір мәселе, ол екі availability zone арасындағы байланысты жақсарту үшін AWS Global Accelerator пайдалану керектігін айтады. AWS Global Accelerator ең жақын аймақтағы S3-ге деректерді жүктеуді жылдамдату үшін қолданылатынын есте ұстаймыз. Содан кейін ең жақын аймақта ол ішкі жаһандық AWS инфрақұрылымы арқылы соңғы AWS аймағына жылдам жетеді. Бұл жүктеу уақытын жылдамдатуға көмектеседі. AWS Global Accelerator cross-availability zone емес, cross-region әрекеттерге қатысты. Сондықтан біз бұл нұсқаны толығымен алып тастаймыз. Екінші нұсқа туралы айтатын болсақ, AWS Organizations сервисіндегі AWS Consolidated Billing пайдалану ұсынылады, бұл әртүрлі департаменттердің барлық AWS аккаунттарын бір негізгі корпоративтік AWS аккаунтында біріктіруге мүмкіндік береді. Бұл AWS аккаунтында қандай аккаунттар бар екенін, аккаунттардың әрқайсысында қандай шығыстар бар екенін көресіз, бірақ бұл аккаунттарды осы аккаунттардың иелері бөлек пайдалана алады. Бұл нұсқа қолайлы сияқты. Басқа нұсқаларды қарастырайық. Үшінші нұсқа AWS Trusted Advisor және AWS Resource Groups Tag Editor пайдалануды ұсынады. AWS Trusted Advisor — инфрақұрылымымыздың best practices-тарға шынымен сәйкес келетінін тексеруге көмектесетін сервис. AWS Resource Groups Tag Editor — ресурстарды топ-топпен белгілеуге мүмкіндік беретін сервис. Бұл әр түрлі департаменттердің ресурстарын бір-бірімен қиылыспауына ешқандай қатысы жоқ. Сондықтан бұл нұсқа толығымен жарамсыз. Төртінші жауапты қарастырайық. Мұнда бір жалпы AWS аккаунтында әр департамент үшін бірнеше түрлі VPC баптау ұсынылады. Әрі қарай, Transit Gateway іске қосып, VPC арасындағы VPN туннельдерін баптау ұсынылады. Мұнда да бұл нұсқа логикалық тұрғыдан мүлдем дұрыс емес. Мұнда VPC арасында VPN туннельдерін пайдалану керектігі айтылған. Шын мәнінде, біз VPC бір-біріне қосу үшін VPC Peering пайдаланамыз. Бұл үшін VPN-ді бөлек баптаудың қажеті жоқ. Тағы бір мәселе, барлық департаменттер бір AWS аккаунтында қалады. Осылайша сіз айдың соңында жалпы шығындарыңызды көресіз. Бұл жерде сұрақтың шарты бойынша әрбір департаментке дербестік беру қажет болды. Енді бесінші жауапты қарастырайық. Мұнда әкімшілер үшін Child Accounts басқаруға мүмкіндік беретіндей cross-account рұқсатты баптау ұсынылады. Яғни, әр департаментке арналған бөлек аккаунттар. Осы талдау негізінде біз екінші және бесінші жауап нұсқаларының ең орынды екенін көреміз. Оның үстіне олар бір-бірін толықтырады. Яғни, бірінші нұсқа барлық аккаунттарды бір ортақ аккаунт астына біріктіреді және барлық шығындардың және пайдаланылатын барлық қолданыстағы AWS аккаунттарының көрінуін жасайды. Ал бесінші нұсқа бізге дүние жүзінде шашыраңқы орналасқан кез келген AWS аккаунтын, департаментін орталықтан басқаруға және оларға қол жеткізуге мүмкіндік береді. Келесі слайдтарда мәтіндік талдау жасалады. Бейнені тоқтатып, егжей-тегжейлі қарауға болады. Осымен Practice Test сұрағын қарастырдық. Назарларыңызға рахмет.
