Қайырлы күн, құрметті студенттер! Зертханалық жұмысты талдау сессиясында көргеніме қуаныштымын. Бүгінгі зертханалық жұмыстың тақырыбы - VPC Peering connection құру. Ендеше, бастайық. Біз AWS Management Console басты парақшасынан бастаймыз. Мұнда бірінші тапсырманың аясында VPC сервисіне бару керек. Ол үшін сервисті іздеу жолағын пайдаланып, VPC деп теруді бастаймыз. Сізге қажет сілтеме іздеу нәтижелерінде бірінші болады. VPC сервисіне өткеннен кейін, біз сол жақтағы шарлау мәзірін пайдаланып, Peering Connections опциясын таңдауымыз керек. Мұнда біз құрылған Peering Connections көрмейміз, жаңасын жасауымыз керек. Ол үшін жоғарғы оң жақтағы Create Peering Connection батырмасын басыңыз. Оны басқаннан кейін біз деректерді енгізу парақшасына бағытталамыз. Мұнда сізге Lab-Peer атауын көрсету керек. Әрі қарай, VPC ID Requester ретінде Lab VPC көрсетуіңіз керек. Парақшаны төмен жылжытамыз. Және VPC ID Accepter ретінде Shared VPC таңдау керек. Содан кейін парақшаның төменгі жағында Create Peering Connection батырмасын басу керек. Біз Peering Connections ресурсының негізгі парақшасына бағытталамыз және осы жерде сәйкес хабарламада Peering Connections жасауды растау керек екенін көреміз. Мұны істейік. Ол үшін Actions батырмасын басып, Accept Request опциясын таңдаңыз. Біз бір аккаунттамыз. Requester мен Accepter үшін аккаунт ID сәйкес. Сондай-ақ, Requester ID - Lab VPC, ал Accepter - Shared VPC екенін көре аласыз. Дұрыс, ұсынысты қабылдай аламыз. Сондықтан, Accept Request батырмасын басыңыз. Оны басқаннан кейін біз Peering Connections тізімі бар парақшаға бағытталамыз, біз Lab-Peer көреміз және күйдің Active екенін көреміз. Зертханалық жұмысымыздың екінші бөлігіне көшеміз және мұнда route table баптауымыз керек. Ол үшін сол жақтағы шарлау мәзірінен Route Tables опциясын таңдаңыз. Біз тиісті парақшаға бағытталамыз. Мұнда Lab Public Route Table табу керек. Оны таңдауымыз керек. Парақшаның төменгі жағында түрлі қойындыларда метадеректері көрсетіледі. Бізді Routes қойындысы қызықтырады. Бізде екі route бар екенін көреміз. Peering Connections үшін қосымша route жасау керек. Ол үшін Edit routes батырмасын басыңыз. Барлық бар route tables осы жерде көрсетіледі. Біріншісі әдепкі, екіншісі Internet Gateway арқылы Интернетке кіруге арналған. Үшіншісі үшін Add route батырмасын басу керек. Әрі қарай, Shared VPC CIDR блогын енгізіңіз және Target ретінде Peering Connections көрсетіңіз. Осыдан кейін өзгертулерді сақтай аламыз. Shared VPC Route table үшін бірдей сол өзгертулер жасалуы керек. Оны таңдауыңыз керек. Мәліметтер парақшасында Routes қойындысын таңдап, Edit routes батырмасын басыңыз. Бұл Route Table үшін тек әдепкі route бар екенін көреміз. Мұнда Add route батырмасын басу керек. Destination ретінде Lab VPC CIDR блогын көрсетуіңіз керек және Target ретінде сол Peering Connections ID көрсетуіңіз керек. Өзгертулерді сақтаңыз. Осымен біз route table-ді баптауды аяқтап, үшінші тапсырмаға көшеміз. Біз үшінші тапсырмаға көшеміз және осы жерде Pering Connections тексереміз. Ол үшін EC2 сервисіне өтіңіз, сервисті іздеу жолағын пайдаланып, EC2 деп теруді бастай аламыз. Іздеу нәтижелерінде бірінші сілтеме бізге қажет сілтеме болып табылады. EC2 сервисінің негізгі парақшасына өткеннен кейін, сол жақтағы шарлау мәзірін пайдаланып, Instances таңдауымыз керек. Мұнда сіз бір инстансты көресіз. Сіз оны таңдап, метадеректерден оның жалпы IP мекенжайын табуыңыз керек. Оны көшіріп алып, браузердегі жаңа қойындыда ашайық. Дерекқорға қосылымды конфигурациялау қажет екендігі туралы хабарлама пайда болғанын көреміз. Мұны істейік және Settings батырмасын басыңыз. Мұнда бізге дерекқордың endpoint қажет болады. Ол үшін AWS Management Console-ге оралып, Amazon RDS сервисіне өтіңіз. Көріп отырғаныңыздай, мұнда бізді қызықтыратын бір ғана инстанс бар. Осы ресурстың негізгі парақшасын ашайық және бірінші Connectivity and Security қойындысында осы дерекқордың endpoint таба аламыз. Оны көшіріп, Endpoint өрісіне қоямыз. Сондай-ақ, Database, Username және Password өрістерін толтыру қажет. Содан кейін Save батырмасын басу керек. Оны басқаннан кейін дерекқордағы жазбалар көрсетіледі. Егер сіз бұл жазбаларды көрген болсаңыз, мен сізді құттықтаймын, біз бәрін дұрыс орнаттық және деректер базасынан деректерді көрдік. Ал бұл деректер базасы мүлдем басқа VPC-де орналасқан. Естеріңізде болса, Shared VPC интернетке қосылу үшін бапталынбаған. Route table-де Internet gateway арқылы Интернетке кіретін route table болмады. Сондықтан дерекқордағы деректерге жетудің жалғыз жолы – Peering Connection арқылы. Біз барлық тапсырмаларды орындадық. Енді біз автоматты бағалау скриптін іске қосуымыз керек. Ол үшін зертханалық жұмыстың негізгі парақшасына оралып, Submit батырмасын басыңыз. Осы скрипттерді іске қосқымыз келетінін растаймыз және біраз уақыттан кейін ұпайларымызды көреміз. Егер сіз максималды балл жинасаңыз, жақсы, құттықтаймын, сіз бәрін дұрыс жасадыңыз. Егер ұпай максимум болмаса, қай тапсырмада ұпайыңыз жеткіліксіз екенін тізімнің төменгі жағына қарап, оны қайтадан орындауға болады. Зертханалық жұмысты аяқтағаннан кейін барлық жүйелерден дұрыс шығу керек. Бұл, ең алдымен, AWS Management Console-іне қатысты. Ол үшін жоғарғы оң жақ бөліктегі пайдаланушы атын басыңыз және ашылмалы тізімде Sign out батырмасын басыңыз. Сондай-ақ, келесі зертханалық жұмысты ашуда қиындықтар болмауы үшін зертханалық жұмысты дұрыс аяқтауымыз керек. Ол үшін, зертханалық жұмыстың парақшасында End lab батырмасын басыңыз және осы қойындыны жабуға болатын хабарды күтіңіз. Осымен зертханалық жұмысты талдау аяқталады. Сұрақтарыңыз болса, хабарласыңыз. Назарларыңызға рахмет. Келесі белсенділіктерде кездескенше.
